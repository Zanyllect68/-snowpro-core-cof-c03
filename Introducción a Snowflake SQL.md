00:21 - 00:30

3. Hojas de trabajo
00:30 - 00:53
Para quienes quieren ejecutar consultas SQL, Snowflake ofrece «Hojas de trabajo», una interfaz para crear y ejecutar consultas SQL. Aquí, como se muestra, podemos crear una hoja de cálculo que contiene la consulta, los resultados y los detalles de la ejecución, como la ejecución de la consulta.

4. Cuadernos
00:53 - 01:07

5. Conexión a Copo de Nieve: Conductores
01:07 - 01:39
Además de su interfaz web, Snowflake se puede utilizar a viajes de controladores, que permiten la comunicación entre sus bases de datos y aplicaciones y herramientas externas como Tableau. Dos de los controladores más comunas son ODBC, que significa «Conectividad de base de datos abierta», y JDBC, o «Conectividad de base de datos Java». Además, hay conectores especializados para lenguajes de programación como Python, que mejor la integración con Snowflake.

6. Conexión a Snowflake: CLI de Snowflake
01:39 - 02:10
La CLI de Snowflake es otra forma de conectar directamente a tu cuenta de Snowflake mediante la línea de comandos. Antes de usar la CLI de Snowflake, tenemos que instalar. Snowflake ofrece versiones para Linux, Windows y Mac. A lo largo de este curso, interactuaremos principalmente a viajes de la CLI de Snowflake, une de vez en cuando haciendo referencia a PostgreSQL para facilitar la comparación.

7. Dialectos de SQL
02:10 - 02:38
Independiente de modo nos conectamos a Snowflake y ejecutivos consultas, siempre usamos Snowflake SQL, una variante de SQL. Snowflake SQL es similar a otras variantes de SQL, como PostgreSQL, T-SQL y MySQL. Estos lenguajes tienen similitudes y diferencias en cuanto a tipos de datos, funciones y sintaxis general.

8. Sintaxis común
02:38 - 03:04
La mayor parte de la sintaxis de Snowflake SQL y PostgreSQL es igual, incluido palabras clave y funciones como `SELECT`, `FROM`, `WHERE`,`GROUPBY`, y muchas otras. ¡Esto significa que, si ya tienes estado trabajando con PostgreSQL, ya puedes elegir todo tipo de consultas con Snowflake SQL!
